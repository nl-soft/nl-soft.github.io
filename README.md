# НЛ-Софт (nl-soft.github.io)

Статический сайт компании, размещённый на GitHub Pages.

## Ветки и процесс изменений

- **main** - прод (то, что видят пользователи сайта).
- **Dev** - рабочая ветка для накопления изменений.

Рекомендуемый workflow:
1) Вносить изменения через feature-ветку от **Dev**.
2) Открывать PR в **Dev**.
3) После проверки/правок - сливать в **Dev**.
4) Периодически делать PR **Dev -> main** для публикации в прод.

## Структура репозитория

- `index.html` - главная (о компании, контакты)
- `experience.html` - опыт/компетенции/ставки
- `requisites.html` - реквизиты
- `policy.html` - политика обработки персональных данных
- `css/` - стили
- `js/` - скрипты
- `vendor/` - локальные зависимости (Bootstrap/jQuery и т.п.)
- `CNAME` - домен для GitHub Pages

## Зависимости

Принцип: **всё локально** (без внешних CDN).

- Bootstrap, jQuery и плагины лежат в `vendor/`.
- Внешние ссылки на библиотеки (CDN) не используем. Если нужен новый UI-компонент - добавляем его в `vendor/` или реализуем без зависимости.

## Как править сайт

Сайт статический - достаточно изменить HTML/CSS/JS файлы.

Рекомендации:
- Для небольших правок: редактируй соответствующий `.html`.
- Для стилей: предпочитай изменения в `css/`, избегай inline-стилей в страницах.
- Для общих элементов (меню/футер) - следи за консистентностью на всех страницах.

## Публикация (GitHub Pages)

Публикация выполняется GitHub Pages.

- Файл `CNAME` должен соответствовать домену.
- Источник публикации (ветка/папка) настраивается в Settings -> Pages.

Проверка после релиза:
- Главная: `/`
- Опыт: `/experience.html`
- Реквизиты: `/requisites.html`
- Политика: `/policy.html`
